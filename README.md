## ما هو البرومبت الرئيسي؟

البرومبت الرئيسي (Master Development Prompt) هو تعليمات تنفيذية متكاملة موجودة في القسم 19 من مواصفة المتطلبات البرمجية (SRS) لتطبيق DataForge AI. صُمّم ليُعطى لمهندس Android أول كفاءة أو لمساعد ذكاء اصطناعي للبرمجة (مثل Claude أو Cursor) ليقوم ببناء التطبيق بالكامل من الصفر حتى الإصدار 1.0 GA — دون الحاجة لطرح أي أسئلة إضافية أو تخمين أي تفاصيل.

## الهدف الأساسي

تحويل مواصفة الـ SRS الطويلة (141 صفحة) إلى أمر عمل عملي قابل للتنفيذ خطوة بخطوة، بحيث يعرف المنفذ بالضبط: **ماذا يبني، بأي تقنية، بأي ترتيب، ومتى يعتبر كل مرحلة منجزة.**

## المكوّنات الرئيسية

- **الهوية والمهمة:** تعريف المنفذ كمهندس Android أول في Quantum Labs، يعمل بـ Kotlin 2.0+ وJetpack Compose وHilt وRoom، مع التزام صارم بـ Clean Architecture.
- **ثوابت المشروع:** اسم التطبيق، المطور، حقوق النشر، minSdk 26، targetSdk 35، اللغتان العربية والإنجليزية، تشفير AES-256-GCM، خوارزمية سلسلة التدقيق SHA-256.
- **الثوابت المعمارية (10 قواعد):** فصل طبقات Clean Architecture، UDF، Compose فقط، Hilt، Room + SQLCipher، offline-first، سجل تدقيق مُسلسل تشفيريًا، RTL كمواطن أول، اختبارات إلزامية.
- **خطة التنفيذ (14 مرحلة، 160 يوم عمل):** من هيكل المشروع إلى نظام التصميم، المحلّلات، محرّك التنظيف، التحقق، الوصفات، الذكاء الاصطناعي، التحليل، الشاشات، التصدير، تقوية الأمان، الأداء، ثم Beta و GA.
- **معايير الكود:** قواعد إلزامية لـ Kotlin وCompose وCoroutines ومعالجة الأخطاء والتعليقات والاختبارات.
- **قائمة مراجعة PR (10 نقاط):** كل ما يجب التحقق منه قبل دمج أي Pull Request.
- **تعريف الإنجاز (9 معايير):** متى تُعتبر أي ميزة منجزة بالكامل.
- **التحقق النهائي (14 سيناريو):** سيناريوهات اختبار شاملة يجب أن يمر عليها التطبيق قبل إعلان GA.

## كيف يُستخدم عمليًا

### الطريقة 1 — مع مساعد ذكاء اصطناعي

انسخ القسم 19 كاملًا → الصقه في Claude أو Cursor أو أي مساعد برمجة → أرفق الأقسام 1-18 كسياق → اطلب: **"ابدأ التنفيذ من Phase 0 وأكمل كل مرحلة بمعايير التحقق".**

### الطريقة 2 — مع مهندس بشري

مهندس جديد ينضم للفريق يقرأ القسم 19 → يبدأ من Phase 0 → ينفّذ → يتحقق → ينتقل للمرحلة التالية، دون الحاجة لسؤال أي شخص.

### الطريقة 3 — لمراجعة الكود

يستخدمه الـ Code Reviewer كـ checklist للتحقق من أن أي Pull Request يلتزم بالمعايير المعمارية ومعايير الكود وتعريف الإنجاز.

## المخرجات النهائية المتوقعة

- تطبيق Android كامل وجاهز للنشر على Google Play (1.0 GA)
- دعم 14 صيغة ملف: CSV, XLSX, PDF, DOCX, JSON, XML, Parquet, SQLite, PPTX, صور، إلخ
- محرّك تنظيف بـ 11 عملية احترافية مع معاينة وتراجع وتدقيق
- ذكاء اصطناعي على الجهاز (Gemma 2B + Phi-3-mini) — بدون إرسال بيانات للسحابة
- سجل تدقيق تشفيري مُسلسل (SHA-256) قابل لإعادة التشغيل والتصدير الموقّع
- واجهة Material Design 3 + glassmorphism بالعربية والإنجليزية (RTL/LTR)
- أمان على مستوى المؤسسات: Keystore + SQLCipher + biometric + panic wipe
- اختبارات شاملة: 85%+ line coverage، 100% critical-path UI، macrobenchmarks
